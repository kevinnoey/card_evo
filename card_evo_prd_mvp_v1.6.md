> 基于 v1.5，新增 **角色选择系统**（双角色：渗透者 + 代码崩溃者）、**代码崩溃者完整卡池（D01-D20，20张）**、**崩溃叠层机制**、**AI训练TCP Bridge**。全部40张卡牌均支持3级进化。

> **最后更新**: 2026-06-11

## 一、原型目标与验证指标

### 1.1 核心验证目标
验证玩家在每回合面对"打出卡牌"与"注入进化"时，是否能感受到有意义的策略抉择，且附带收益能否有效缓解前期生存压力，并形成"短期生存与长期投资"的拉扯感。同时验证**双角色差异化**是否能提供截然不同的策略体验。

### 1.2 成功衡量指标
**人工反馈指标**
- **策略清晰度**：玩家能否在3局游戏内理解EP注入与进化的规则？
- **决策占比**：玩家在一局战斗中，选择"注入EP"的回合数是否达到总回合数的30%以上。
- **心流体验**：玩家打出Lv.3卡牌引发"进化绽放"时的情绪反馈是否正面。
- **认知负荷**：玩家是否觉得同时计算EP、敌方意图和卡牌层数过于疲劳？
- **角色偏好**（v1.6新增）：玩家对渗透者和代码崩溃者的选择比例及满意度。
- **崩溃策略理解**：崩溃者玩家能否在2局内理解"叠层→爆发→清零"的节奏？

**系统自动记录的数据点**
- **注入/出牌比**：每回合平均注入次数与打出卡牌次数的比值。
- **进化达成率**：卡牌进化至Lv.2/Lv.3的场次占比。
- **溢出利用率**：全局EP储备条被充满并触发免费升级的场均次数。
- **奖励选择分布**：每种奖励被选择的次数占比。
- **重选利用率**：奖励重选按钮被使用的频率。
- **地图路径选择**：各节点类型被选择的频率。
- **事件选择分布**：各事件选项被选择的次数占比。
- **商店购买分布**：各商品被购买的次数占比。
- **角色选择率**（v1.6新增）：两个角色被选择的次数占比。
- **崩溃层数利用**（v1.6新增）：崩溃者每场战斗中崩溃叠层达到满层的频率。

## 二、范围界定（Scope）

### 2.1 包含内容 (In Scope)
- 基础回合制战斗框架（抽牌、弃牌、手牌上限5张、敌方意图展示）。
- **登录界面**：双阶段流程——背景图片+赛博朋克风格开始按钮 → 角色选择（渗透者/代码崩溃者）。
- 完整的EP经济系统（消费、注入、屏障附带收益、储备池阶段性增益）。
- **全部40张卡牌均支持3级进化**（Lv.1→Lv.2 需8EP，Lv.2→Lv.3 需20EP）。
- 卡牌进化状态机与"进化绽放"效果（全屏闪白、卡牌抖动、金色粒子特效）。
- 卡牌扇形弧形手牌排列（动态角度计算、层级堆叠、悬停弹出）。
- **2名可选角色**：渗透者（HP 70, 穿甲/屏障/进化）+ 代码崩溃者（HP 55, 崩溃叠层/终结爆发）。
- **崩溃叠层机制**（v1.6新增）：敌人最大5层崩溃，每层+3攻击伤害，崩溃者独有。
- 40张完整卡池（渗透者C01-C20 + 崩溃者D01-D20，每张均有3级进化数据）。
- **9名敌人**（第一章6名 + 第二章3名）+ 3个增强变体。
- **Roguelite 地图系统**：程序化章节地图、6种节点类型、事件/商店/休息系统。
- **数据碎片经济**（战斗/事件获得，商店消费）。
- **全局行动日志**（跨场景持久化，实时显示）。
- **战后奖励系统**：8种奖励类型，含一次重选机会。
- **「核心突破」章节过渡奖励**：5种突破协议。
- **角色信息面板**：点击"?"图标查看技能与机制说明。
- **角色/敌人立绘**：带霓虹发光边框、呼吸光效、扫描线、能量粒子、闲置动画。
- **HUD霓虹装饰**：血条和进度条外框带发光边框与角标装饰。
- **AI训练TCP Bridge**（v1.6新增）：ExternalControlService C# Autoload，支持外部AI程序控制游戏。

- 义体系统（核心模块与插件）。
- 赛博精神病（乱码/Glitch）退化系统。
- 局外成长与元进度系统（跨Run永久升级）。
- 高品质美术与音效（全程灰盒UI + 程序化生成）。
- 第三章及后续内容。
- 多人模式。
- 第三名及更多角色。

## 三、核心系统需求规范

### 3.1 战斗循环基础
- **分辨率**：1920 × 1080。
- **回合流程**：回合开始（抽5张牌，恢复3点可分配EP，清除屏障和格挡）→ 玩家操作期 → 回合结束（剩余EP以粒子动画注入储备池）→ 敌人行动期 → 下一回合。
- **牌库逻辑**：起始牌库10张。抽牌堆耗尽时，弃牌堆洗牌重新变为抽牌堆。
- **进度保留**：卡牌被弃掉时，其已累积的EP注入进度完全保留（按进化家族 `evolution_family` 追踪）。
- **手牌上限**：手牌最多5张（可通过「超限跃迁」协议提升至6张）。超出的牌必须由玩家点击选择弃掉。

### 3.2 EP交互与反馈机制

**打出卡牌**
- 左键点击手牌，消耗卡牌费用（0-3 EP），执行卡牌效果。

**注入进化**
- **操作**：右键点击手牌，消耗1点EP注入。所有卡牌均可注入。
- **即时附带收益**：每次成功注入，获得 **2点屏障**。
- **进度显示**：卡牌下方绿色进度条 + 数字显示"已注入/需注入"。
- **悬停Tooltip**：鼠标悬停展示 Lv.1/Lv.2/Lv.3 完整效果。
- **进化家族键值追踪**：EP注入进度以 `evolution_family` 为键存储。
- **跨战斗进化等级追踪**：`family_max_level` 字典记录最高进化等级。

**崩溃叠层机制（v1.6新增 — 代码崩溃者独有）**
- **崩溃层数**：存在于敌人实体上的状态（`enemy.crash_stacks`），最大 **5层**。
- **伤害加成**：每层崩溃使玩家下次攻击额外 **+3伤害**（`CRASH_DAMAGE_PER_STACK = 3`）。
- **施加方式**：通过崩溃诱导(D07)、崩溃脉冲(D01)、崩溃协议(D09)等卡牌施加。
- **层数消耗**：部分攻击牌（如零日漏洞D11、数据收割D10）命中后消耗崩溃层数触发额外效果。
- **与进化共存**：崩溃者同样支持EP注入进化系统。

**溢出储备池**
- 点击"结束回合"时，剩余EP以粒子动画飞入储备条。
- 储备池上限 15（可通过「涌动核心」协议扩展至20）。
- **能量护盾**：储备池每累计5点，获得1层能量护盾（永久+1全伤害减免）。
- **免费升级**：储备池满15点时，可将手牌中一张程序牌直接提升1级。

### 3.3 卡牌进化状态机
- **Lv.1**：基础形态。进化需 8 EP。
- **Lv.2**：累计注入达 8 EP 时自动转变。进化需 20 EP。
- **Lv.3（终极）**：累计注入达 20 EP 时自动转变，触发 **"进化绽放"**。
- **溢出处理**：进化时多余EP注入储备池。

**进化绽放效果**
- 触发瞬间：卡牌白色双脉冲闪烁 + 金色粒子爆发 + 全屏白色闪光 + 卡牌抖动。
- 费用立即变为 0（`cost_override = 0`）。
- 允许玩家额外打出一次。

### 3.4 手牌弧形排列（扇形）
- 手牌沿虚拟圆的顶部弧线排列。
- 参数：水平半径 R=2000，垂直半径 R_Y=1200，每卡角度 0.05rad，最大展开 0.4rad。
- 悬停效果：z_index=100 置顶，缩放1.12×，上浮15px。
- 卡牌尺寸：240 × 330px。

## 七、技术实现备注

- **引擎**：Godot 4.6.2 Mono（D3D12 Forward+渲染器，Jolt物理）。
- **架构**：双场景架构 — `map_screen.tscn`（地图导航）+ `battle.tscn`（战斗）。
- **卡牌弧形**：R=2000，R_Y=1200，每卡 0.05rad，最大 0.4rad。
- **进度追踪**：EP注入进度按 `evolution_family` 键值存储，与卡牌实例解耦。
- **跨战斗进化持久化**：`family_max_level` 追踪最高进化等级，战斗开始时重建牌库。
- **角色选择持久化**：`RunRewardState.selected_character` 存储角色ID，影响初始牌库和HP。
- **崩溃叠层实现**：`Enemy.crash_stacks` 变量 + `MAX_CRASH_STACKS=5` + `CRASH_DAMAGE_PER_STACK=3`。
- **伤害计算顺序**：能量护盾减免 → 屏障吸收 → 格挡吸收。敌方伤害需过ICE减免。
- **数据流**：CardDatabase → CardUI → EPManager → BattleManager → BattleHUD。信号驱动架构。
- **BattleManager 关键状态变量**：
  - `next_attack_bonus: int` — D08 弱点扫描：本回合下次攻击+3（使用后清零）
  - `player_damaged_last_turn: bool` — D06 回滚：记录上回合是否受HP伤害
  - `barrier_reduction_next_turn: int` — 数据腐化体污染：下回合玩家屏障获得减少
  - `extra_play_available: bool` — 进化绽放额外行动标志
  - `fast_mode: bool` — AI训练快速模式（跳过动画延迟）
- **敌人AI系统**：支持 random、cycle、weighted_random、smart、multi_phase。
- **多阶段Boss**：Phase切换在HP阈值（50%）触发。
- **TCP Bridge**：ExternalControlService.cs 监听端口 47077，通过 BridgeAPI GDScript 节点转发命令。

### 7.1 Roguelite 系统架构

- **场景流转**：`login_screen.tscn` → `map_screen.tscn` → `battle.tscn` → `map_screen.tscn`（循环）→ 通关/失败。
- **RunRewardState 单例**（Autoload）作为中央 Run 管理器，核心变量包括：
  - `selected_character` — 角色ID（infiltrator/crasher）
  - `player_deck` / `player_hp` / `player_max_hp` — 牌库与HP持久化
  - `card_ep_by_family` / `family_max_level` — 进化进度持久化
  - `reserve_pool` / `data_fragments` — 储备池进度与碎片货币
  - `map_seed` — 地图生成种子（支持可复现地图）
  - `permanent_ep_bonus` — 永久EP/回合加成（黑市/祭坛事件）
  - `next_battle_enemy_bonus_strength` — 下场敌人力量加成（黑市举报选项）
  - `next_battle_temp_bonus` — 下场战斗临时加成（屏障/格挡/EP）
- **MapGenerator** 生成 4层节点（3普通+1Boss）。
- **ActionLog** 单例存储整个Run的操作记录。

### 7.2 AI训练 Bridge 架构（v1.6新增）

```
[Python训练脚本] --TCP--> [ExternalControlService.cs] --Call--> [BridgeAPI.gd] --信号--> [BattleManager]
```

- **命令类型**：`ping` / `get_snapshot` / `execute_action` / `set_fast_mode`
- **Fast Mode**：跳过动画延迟，加速数据采集
- **端口**：47077（可通过 `CARD_EVO_BRIDGE_PORT` 环境变量配置）
