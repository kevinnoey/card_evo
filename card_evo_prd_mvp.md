# 《进化核心》原型验证阶段（MVP）产品需求文档（修订版）

## 一、原型目标与验证指标

### 1.1 核心验证目标
验证玩家在每回合面对“打出卡牌”与“注入进化”时，是否能感受到有意义的策略抉择，且附带收益能否有效缓解前期生存压力，并形成“短期生存与长期投资”的拉扯感。

### 1.2 成功衡量指标（Playtest 反馈 + 自动采集数据）
**（1）人工反馈指标**
- **策略清晰度**：玩家能否在3局游戏内理解EP注入与进化的规则？
- **决策占比**：玩家在一局战斗中，选择“注入EP”的回合数是否达到总回合数的30%以上。
- **心流体验**：玩家打出Lv.3卡牌引发“进化绽放”时的情绪反馈是否正面（爽感）。
- **认知负荷**：玩家是否觉得同时计算EP、敌方意图和卡牌层数过于疲劳？

**（2）系统自动记录的数据点**
- **注入/出牌比**：每回合平均注入次数与打出卡牌次数的比值。
- **进化达成率**：卡牌进化至Lv.2/Lv.3的场次占比，以及进化后当回合立即被打出的比例。
- **溢出利用率**：全局EP储备条被充满并触发免费升级的场均次数。

## 二、范围界定（Scope）
为了在3-4个月内快速跑通原型，我们必须严格控制开发范围。

### 2.1 包含内容 (In Scope)
- 基础回合制战斗框架（抽牌、弃牌、敌方意图展示）。
- 完整的EP经济系统（消费、注入、附带收益、全局进度条及阶段性微型增益）。
- 卡牌三级进化系统与“进化绽放”效果。
- 1名测试角色（基础能力设定）。
- 20张精简卡池（包含0费卡以增加策略维度）。
- 2种敌人（1个具有意图随机性的杂兵，1个高生存压力精英）。
- 线性战斗序列（无地图探索，打完一场直接进入下一场）。

### 2.2 排除内容 (Out of Scope - 严禁在此时开发)
- 随机地图生成与节点选择（Map UI）。
- 义体系统（核心模块与插件）。
- 赛博精神病（乱码/Glitch）退化系统。
- 局外成长与元进度系统。
- 商店与事件系统。
- 高品质美术与音效（全程使用灰盒UI、免费素材和占位图）。

## 三、核心系统需求规范

### 3.1 战斗循环基础
- **回合流程**：回合开始（抽5张牌，恢复3点可分配EP） → 玩家操作期 → 回合结束（弃掉未使用的手牌，清空未分配的EP） → 敌人行动期 → 下一回合。
- **牌库逻辑**：起始牌库10张，抽牌堆耗尽时，弃牌堆洗牌重新变为抽牌堆。**关键规则：卡牌因回合结束被弃掉时，其已累积的EP注入进度完全保留。**

### 3.2 EP交互与反馈机制（最核心功能）
玩家通过拖拽或点击与手牌交互。

- **打出卡牌**：消耗对应EP，执行卡牌基础效果。
- **注入进化**：
  - **操作**：玩家右键点击（或长按）手牌，消耗1点EP注入该卡牌。
  - **即时附带收益**：每次成功注入，玩家立刻获得 **2点屏障**。该屏障作为临时生命值，**可保留至下回合开始**（即能抵挡敌人本回合或下回合的首次伤害判定），并在下回合结束时若未消耗则消失。
  - **进度显示**：卡牌UI上必须有清晰的进度槽，并 **以数字显示“距下级还需 X EP”**（如 `[▓▓░░░░░░] 3/8`）。

- **溢出储备池**：
  - 玩家点击“结束回合”时，若仍有剩余EP，自动注入屏幕右上角的“EP储备条”（上限15）。
  - **微型增益（关键改动）**：储备条每累计达到5点，玩家立即获得1层“能量护盾”（可叠加，每层提供1点全伤害减免，战斗结束后消失）。达到15点时，消耗全部15点，为玩家提供一次“免费升级”：将手牌中任意一张程序牌直接提升1级（若已Lv.3则无效），然后储备条归零。
  - **反馈动画**：回合结束时，剩余EP需以光点动画流入储备条，让因果关系清晰可见。

### 3.3 卡牌进化状态机
- 卡牌数据结构需支持动态变化：
  - **Lv.1**：基础形态。
  - **Lv.2**：当累计注入达8 EP时自动转变，卡牌名称、描述、数值实时更新。
  - **Lv.3（终极）**：当累计注入达20 EP时自动转变，触发 **“进化绽放”**。
- **进化绽放效果明确规则**：
  - 触发时，该卡牌本回合费用立即变为0，并 **允许玩家在当前操作完成后额外打出一次**（不消耗常规行动机会）。若玩家不立即打出，该牌保留0费状态直至下回合结束，之后恢复正常Lv.3费用。
  - 触发瞬间应有全屏闪白及卡牌放大特效（低保真下可用色块闪烁+卡牌抖动实现），强化爽感反馈。

## 五、UI/UX 原型要求（低保真/灰盒标准）
信息层级必须绝对清晰，所有动态反馈必须可视：

1. **左上角**：玩家HP条、当前可用EP（3个发光点阵 + 阿拉伯数字显示，如“◆◆◇ 2”）。
2. **右上角**：全局EP储备条（进度条形式 0/15），每达到5/10/15刻度时有高亮标记，旁边显示当前能量护盾层数。
3. **敌人头顶**：巨大且显眼的意图图标（剑=攻击，盾=防御，骷髅=重击）及具体数值。
4. **手牌区域**：
   - 卡牌费用在左上角。
   - 可进化的程序牌，边框用**亮蓝色**区分，下方有高亮进度条（已注入部分为荧光绿，未注入为深灰），进度条末端显示“3/8”等数字。
   - 当玩家悬停在程序牌上时，通过 Tooltip（提示框）清晰展示其 Lv.2 和 Lv.3 的完整效果，并标注“弃牌保留进度”。
5. **回合结束反馈**：点击结束回合时，若有剩余EP，以粒子流形式从EP点阵飞入右上角储备条，并伴有轻微音效（可用单音效提示）。
