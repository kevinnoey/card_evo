## 一、原型目标与验证指标

### 1.1 核心验证目标
验证玩家在每回合面对"打出卡牌"与"注入进化"时，是否能感受到有意义的策略抉择，且附带收益能否有效缓解前期生存压力，并形成"短期生存与长期投资"的拉扯感。

### 1.2 成功衡量指标
**人工反馈指标**
- **策略清晰度**：玩家能否在3局游戏内理解EP注入与进化的规则？
- **决策占比**：玩家在一局战斗中，选择"注入EP"的回合数是否达到总回合数的30%以上。
- **心流体验**：玩家打出Lv.3卡牌引发"进化绽放"时的情绪反馈是否正面。
- **认知负荷**：玩家是否觉得同时计算EP、敌方意图和卡牌层数过于疲劳？

**系统自动记录的数据点**
- **注入/出牌比**：每回合平均注入次数与打出卡牌次数的比值。
- **进化达成率**：卡牌进化至Lv.2/Lv.3的场次占比。
- **溢出利用率**：全局EP储备条被充满并触发免费升级的场均次数。
- **奖励选择分布**：每种奖励被选择的次数占比（验证奖励多样性）。
- **重选利用率**：奖励重选按钮被使用的频率（验证玩家对初始选项的不满意度）。
- **地图路径选择**：各节点类型被选择的频率（验证地图策略深度）。
- **事件选择分布**：各事件选项被选择的次数占比。
- **商店购买分布**：各商品被购买的次数占比。

## 三、核心系统需求规范

### 3.1 战斗循环基础
- **分辨率**：1920 × 1080。
- **回合流程**：回合开始（抽5张牌，恢复3点可分配EP，清除屏障和格挡）→ 玩家操作期 → 回合结束（剩余EP以粒子动画注入储备池）→ 敌人行动期 → 下一回合。
- **牌库逻辑**：起始牌库10张。抽牌堆耗尽时，弃牌堆洗牌重新变为抽牌堆。
- **进度保留**：卡牌被弃掉时，其已累积的EP注入进度完全保留（按进化家族 `evolution_family` 追踪，不依赖实例ID）。
- **手牌上限**：手牌最多5张（可通过「超限跃迁」协议提升至6张）。超出的牌必须由玩家点击选择弃掉（选牌期间禁止出牌、注入和结束回合，所有卡牌短暂红色闪烁提示）。

### 3.2 EP交互与反馈机制

**打出卡牌**
- 左键点击手牌，消耗卡牌费用（0-3 EP），执行卡牌效果。

**注入进化**
- **操作**：右键点击手牌，消耗1点EP注入。所有卡牌均可注入（不仅限于程序牌）。
- **即时附带收益**：每次成功注入，玩家立刻获得 **2点屏障**（临时HP，下回合开始时消失，优先于格挡吸收伤害）。
- **进度显示**：卡牌下方绿色进度条 + 数字显示"已注入/需注入"（如 `3/8`）。进度条满时变为金色脉冲。
- **悬停Tooltip**：鼠标悬停任意卡牌时展示 Lv.1/Lv.2/Lv.3 完整效果 + "弃牌保留进度"。程序牌（PROGRAM类型）边框为亮蓝色(`#00BFFF`)，其余卡牌边框颜色对应其类型色。
- **进化家族键值追踪**：EP注入进度以 `evolution_family` 为键存储在 `card_ep_by_family` 字典中，确保同一家族的卡牌实例共享进度。
- **跨战斗进化等级追踪**：`family_max_level` 字典记录每个家族达到的最高进化等级（1/2/3），在战斗间重建牌库时应用。

**溢出储备池**
- 点击"结束回合"时，剩余EP以**金色光点粒子动画**飞入屏幕中央的EP储备条（每个EP一个粒子，间隔0.08s）。
- 储备池上限 15（可通过「涌动核心」协议扩展至20），进度条显示 0/15，刻度标记在 5/10（橙色竖线）。
- **能量护盾**：储备池每累计5点（5/10/15），玩家获得1层能量护盾（永久+1全伤害减免，战斗结束后消失）。同一阈值不重复触发（`milestones_reached` 追踪）。
- **免费升级**：储备池满15点时，消耗全部15点，玩家可将手牌中一张程序牌直接提升1级（Lv.3无效），然后归零。（注：当前版本信号已触发，消息已显示，选择UI待完善）

### 3.3 卡牌进化状态机
- **Lv.1**：基础形态。进化需 8 EP。
- **Lv.2**：累计注入达 8 EP 时自动转变，名称/描述/数值实时更新。进化需 20 EP。
- **Lv.3（终极）**：累计注入达 20 EP 时自动转变，触发 **"进化绽放"**。
- **溢出处理**：进化时多余EP注入储备池（如家族已有22EP，Lv.2→Lv.3消耗20EP，溢出2EP进入储备池）。

**进化绽放效果**
- 触发瞬间：卡牌白色双脉冲闪烁 + 金色粒子爆发(16粒) + 全屏白色闪光(0→0.8→0) + 卡牌抖动(8帧震荡)。
- 费用立即变为 0（`cost_override = 0`）。
- 允许玩家在当前操作完成后**额外打出一次**（不消耗常规行动机会，`extra_play_available = true`）。
- 若不打，保留0费状态至下回合结束，之后恢复正常Lv.3费用。

### 3.4 手牌弧形排列（扇形）
- 手牌沿虚拟圆的**顶部弧线**排列，圆心位于手牌区下方。
- 公式：`x = cx + R·sin(θ)`，`y = -R_Y·(1 - cos(θ))`，rotation = θ。
- 参数：水平半径 R=2000，垂直半径 R_Y=1200，圆心偏移 cx = 手牌区中点-200，每卡角度 ANGLE_PER_CARD=0.05rad，最大展开 MAX_SPREAD=0.4rad。
- 边界约束：x 坐标 clamp 在半张卡宽(120px)到手牌区右边界之间。
- 层级：中央牌 z_index 最高，向两侧递减。
- 悬停效果：z_index=100 置顶，缩放1.12×，上浮15px，阴影放大(16px + 蓝色辉光)。
- 卡牌尺寸：240 × 330px，pivot_offset 默认。

### 4.3 登录界面
- **背景**：使用 `登陆界面.png` 图片，expand_ignore_size 拉伸至 1920×1080，亮度调制 1.0（不再压暗）。
- **开始按钮**："开 始 游 戏"（350×100, 40pt, 白色），赛博朋克霓虹风格——默认紫色 `#6B3BFF` 边框+发光阴影，悬停金色 `#FFD700` 边框+增强阴影，按下橙色 `#FFA000` 边框+最强阴影。
- **按钮动画**：alpha 0.85↔1.0 脉冲循环（周期3s）。
- **按钮装饰**：网格叠加层 (580×90, `#4A3BFF` 6%透明度)。
- **操作提示**："左键出牌 | 右键注入 | 结束回合注入储备池"（20pt, `#9999CC`，位于按钮下方）。
- 点击"开始游戏"→ 淡入角色选择画面（双角色卡片：渗透者 / 代码崩溃者）。
- 选择角色后点击"确认选择"→ 切换到 `map_screen.tscn`（进入地图而非直接战斗）。

### 4.4 角色信息面板
- **触发**：点击玩家名称旁的"?"图标。
- **样式**：全屏半透明黑色遮罩(z_index=200) + 居中面板(1000×700, `#0A0E27` 背景, `#00F0FF` 2px边框, 圆角16px)。
- **标题**："渗透者"（36pt, `#00F0FF`，居中）。
- **属性行**："HP: 70  |  EP/回合: 3  |  手牌上限: 5"（22pt, `#FFD700`，居中）。
- **机制说明**（8条，左列名称 `#00BFFF` 18pt + 右列描述 `#CCCCCC` 16pt）：
  1. 打出卡牌 — 左键点击手牌，消耗 EP 执行卡牌效果
  2. 注入 EP 进化 — 右键点击任意卡牌，消耗 1 EP 注入进度
  3. 屏障 — 每次注入 EP 获得 2 点屏障，吸收伤害优先于格挡，下回合消失
  4. 格挡 — 使用技能牌获得，吸收本回合伤害，下回合消失
  5. 能量护盾 — 储备池达到 5/10/15 时获得，每层永久减免 1 点所有伤害
  6. 进化绽放 — 卡牌达到 Lv.3 时触发：费用变为 0，可额外行动一次
  7. EP 储备池 — 回合结束时剩余 EP 自动注入储备池（上限 15），达到 5/10 获得护盾，15 可免费升级
  8. 弃牌保留进度 — 卡牌的 EP 注入进度在弃牌/洗牌后完全保留
- **返回按钮**："返回战斗"（240×52, 22pt, `#3B3B6B` 背景, `#00F0FF` 边框）。

### 4.5 立绘特效系统（PortraitFX）
- **呼吸光晕**：霓虹边框 `shadow_size` 在 6~18 之间正弦波循环（1.5s周期），角标亮度同步变化。
- **闲置动画**：立绘容器 x 轴 ±3px 摇摆（周期 1s+2s+1s），y 轴 -4~+3px 浮动（周期 1.5s+1.5s+1s），scale 1.0~1.025 呼吸缩放（周期 2s+2s）。播放 6~10s 后暂停 0.8~1.8s，模拟自然停顿。
- **能量粒子**：边框边缘随机位置生成 3~5 个彩色粒子（3~6px方块），上飘 40~80px 后淡出（1.2~1.8s），间隔 1.5~2.8s 循环。
- **扫描线**：对角方向光条（18px宽，10%透明度）从上到下扫过立绘区域（2.5s），间隔 6s 再次扫描。
- **受击反应**：立绘水平抖动（6帧，±4px）、闪色（红/蓝/青/金）、粒子爆发（8粒向外散射）。
- **攻击突进**：立绘向目标方向突进20px后弹回（TRANS_BACK缓动）。

## 七、技术实现备注

- **引擎**：Godot 4.6.2 Mono（D3D12 Forward+渲染器，Jolt物理）。
- **架构**：
  - **v1.5 及以前**：BattleManager 根节点创建所有子节点和信号连接。
  - **v1.6**：双场景架构 — `map_screen.tscn`（地图导航）+ `battle.tscn`（战斗），通过 `RunRewardState` Autoload 单例在场景间传递状态。
- **卡牌弧形**：水平半径 R=2000，垂直半径 R_Y=1200，圆心cx=手牌区中点-200。每卡角度0.05rad，最大总展开0.4rad。x坐标clamp至[120, 手牌区宽-120]。
- **进度追踪**：EP注入进度按 `evolution_family` 键值存储在 `card_ep_by_family` 字典中，与卡牌实例解耦，支持弃牌/洗牌后恢复。
- **跨战斗进化持久化**：
  - `family_max_level` 字典追踪每个家族达到的最高进化等级（1/2/3）。
  - 战斗开始时，`_build_evolved_deck()` 从 `RunRewardState.player_deck` + `family_max_level` 重建牌库，确保进化后的卡牌正确出现。
  - `card_ep_by_family` 和 `family_max_level` 在战斗结束后通过 `RunRewardState.sync_from_ep_manager()` 持久化。
- **进化触发**：奖励系统注入EP通过 `inject_to_card_free()` 方法（而非直接操作 `card_ep_by_family`），确保进化阈值检查与 `_evolve_card()` 正确触发。
- **进化溢出**：进化触发时，超出所需EP的部分自动注入储备池。
- **主题防泄漏**：每次 refresh_display 重新设置 theme_override，避免残留。
- **Tween管理**：悬停tween、消息渐隐tween、进化脉冲tween均存储引用并kill后重建，避免竞争。
- **出牌动画清理**：_animate_play_card 的 tween.finished 连接 card.queue_free，确保动画完成后释放。
- **伤害计算顺序**：能量护盾减免（永久全减）→ 屏障吸收（临时）→ 格挡吸收（本回合）。敌方伤害还需过ICE减免（穿甲可抵消）。
- **数据流**：CardDatabase（静态数据）→ CardUI（显示实例）→ EPManager（进度状态）→ BattleManager（协调）→ BattleHUD（UI更新）。信号驱动架构覆盖全部状态变更。
- **敌人AI系统**：Enemy类需支持多种AI类型（random、cycle、weighted_random、smart、multi_phase），通过 `_ai_type` 字段区分，`_get_intent()` 方法根据类型返回不同行为。智能AI（虚空信标）的判定时机为**敌人回合开始时**，读取玩家当前手牌数量决定行动。
- **多阶段Boss**：湮灭协议/格式化巨兽的Phase切换在BattleManager中监听HP阈值（50%），触发 `_phase_transition_effect()`（闪红+抖动+粒子爆发）并重置AI行为树。
- **弃牌EP恢复**：卡牌被弃掉时，其 `injected_ep` 字段记录的已注入EP会被返还给玩家，确保弃牌不会造成EP损失。

### 7.1 Roguelite 系统架构（v1.6新增）

- **场景流转**：`login_screen.tscn` → `map_screen.tscn` → `battle.tscn` → `map_screen.tscn`（循环）→ 通关/失败。
- **RunRewardState 单例**（Autoload）作为中央 Run 管理器，存储：
  - `run_active: bool` — 是否有活跃的游戏Run
  - `current_chapter: int` — 当前章节（1/2）
  - `player_deck: Array` — 玩家牌库（base card_id 数组）
  - `player_hp/max_hp` — 跨场景HP持久化
  - `player_energy_shield` — 跨战斗能量护盾持久化
  - `card_ep_by_family/family_max_level` — 进化进度持久化
  - `reserve_pool` — 储备池进度持久化
  - `data_fragments` — 数据碎片货币
  - `current_map_data` — 当前章节地图数据
  - `completed_nodes` — 已完成节点列表
  - `pending_battle` — 待进入战斗的配置
  - `chapter_protocols` — 已激活的突破协议
  - `permanent_ep_bonus` — 永久EP/回合加成
- **地图生成**：`MapGenerator` 类（RefCounted）生成4层节点（3普通+1Boss），使用种子确保可复现性。
- **全局行动日志**：`ActionLog` 单例（Autoload）存储整个Run的操作记录，战斗和地图场景的日志面板都从该单例读取数据，支持跨场景回放。
- **场景切换数据传递**：
  - 进入战斗前：`RunRewardState.pending_battle` 设置 enemy_key/node_type 等信息
  - 战斗结束后：`RunRewardState.sync_from_ep_manager()` / `sync_hp_from_player()` 回写状态
  - 返回地图后：`MapScreen._handle_return_from_battle()` 读取结果，标记节点完成

## 十、战后奖励系统

> 为提升战斗间的策略深度与成长爽感，在 Roguelite 地图系统中，每次战斗胜利后，玩家可从三个随机奖励中选择其一。v1.6 扩展至 **8 种奖励类型**，并包含一次重选机会。

### 10.1 触发时机与流程

- **触发点**：敌人 HP ≤ 0 时，播放死亡动画与胜利特效，随后弹出奖励选择面板。
- **碎片奖励**：战斗胜利后自动获得数据碎片（普通战斗20-30，精英战斗40-50）。
- **可选范围**：每次战斗胜利后弹出奖励选择。
- **面板行为**：
  - 全屏半透明遮罩（`#000000`, opacity 0.8）+ 居中奖励面板（1100×600, `#0A0E27` 背景, `#FFD700` 2px 边框, 圆角 20px）。
  - 面板标题："战斗胜利 — 选择一项增强"。
  - 三个奖励卡片横向排列（240×320, 间距 40px），鼠标悬停时卡片缩放 1.05 倍，边框变为青色 `#00F0FF`。
  - 点击任一卡片后，播放金色粒子爆发（12 粒）与全屏短闪（白色 0.1s），关闭面板（淡出 0.2s + 缩放 0.95），返回地图界面。
- **重选机制**：面板下方显示"🔄 重新选择一次"按钮，点击后从8种奖励中重新随机抽取3个替换当前卡片，按钮变为禁用状态"✓ 已重新选择"。每个战斗胜利仅可使用1次。
- **信息按钮**：面板左上角"?"按钮，点击弹出全部8种奖励的详细说明面板。
- **跳过处理**：玩家必须选择一个奖励才能继续，无"跳过"选项。

### 10.2 奖励类型库（8 种核心类型）

奖励效果分为 **永久效果**（持续整个Run）和 **临时效果**（仅下一场战斗生效）。所有效果通过 `RunRewardState`（Autoload 单例）存储，战斗开始/结束时应用或清除。

| 奖励名称 | 类型 | 效果描述 | 可叠加性 |
|---------|------|---------|----------|
| **进化模块** | 永久 | 显示所有卡牌家族缩略全尺寸UI（5列×2行网格，CardUI 0.7缩放）。点击后为卡牌家族增加 **8 EP** 注入进度。若累计达到8EP/20EP则触发进化。 | 可多次作用于不同卡牌，或同一卡牌分次注入 |
| **能源核心** | 永久 | 立即增加 **5 点 EP 储备池进度**。若触发 5/10/15 阈值，正常给予能量护盾层数。 | 可叠加 |
| **战术协议** | 临时（下一场） | 下一场战斗开始时，获得 **6 点屏障** + **2 点格挡**。 | 多次选择则数值累加 |
| **超频芯片** | 临时（下一场） | 下一场战斗中，每回合可分配 EP 增加 **1 点**（最高不超过 6）。 | 可叠加，每次 +1，最高 6 EP/回合 |
| **纳米修复** | 永久 | 立即恢复 **25 点 HP**。 | 可叠加（每次恢复25HP，不超过最大HP） |
| **基因强化** | 永久 | 永久增加 **10 点最大HP**，并恢复 **10 点 HP**。 | 可叠加，每次 +10 最大HP + 10HP恢复 |
| **随机卡牌**（v1.6新增） | 永久 | 从所有20张基础卡牌中随机获得一张 Lv.1 卡牌，加入牌库。 | 可叠加（牌库可超过10张） |
| **复制卡牌**（v1.6新增） | 永久 | 显示当前牌库中的卡牌家族网格，选择一张复制加入牌库。 | 可叠加 |

### 10.3 奖励随机规则

- 每次胜利后，从上述 8 种奖励中随机抽取 **3 种不同** 的奖励作为本次选项。
- 每种奖励的文本描述根据当前状态动态调整（例如"进化模块"会提示"选择一张卡牌 +8 EP"；若玩家所有卡牌均已 Lv.3，则自动变更为"获得 +5 储备池"）。
- **重选机制**：玩家每次战斗胜利可点击"🔄 重新选择一次"按钮一次，重新从8种奖励中随机抽取3个替换当前选项。

### 11.3 崩溃叠层机制（Crash Stack）

**代码崩溃者**独有的核心机制：
- **崩溃层数**：敌人实体上的状态，最大 **5 层**（`MAX_CRASH_STACKS = 5`）。
- **伤害加成**：每层崩溃使玩家下次攻击额外 **+3 伤害**（`CRASH_DAMAGE_PER_STACK = 3`）。
- **层数消耗**：部分攻击牌命中后会消耗全部崩溃层数（触发额外效果后清零）。
- **层数叠加**：通过崩溃诱导、崩溃脉冲等卡牌施加崩溃层数。
- **与进化系统共存**：崩溃者同样支持 EP 注入进化（Lv.1→Lv.2 需 8EP，Lv.2→Lv.3 需 20EP）。

## 十三、AI训练 Bridge（实际实现新增）

> 项目包含完整的 TCP Bridge 架构，支持外部 AI 训练程序通过 JSON 协议与游戏交互。

### 13.1 ExternalControlService（C# Autoload）

- **端口**：47077（可通过环境变量 `CARD_EVO_BRIDGE_PORT` 配置）。
- **协议**：一次 TCP 连接 = 一个 JSON 请求 + 一个 JSON 响应。
- **命令类型**：
  - `ping` → 返回 `{"ok":true, "message":"pong"}`
  - `get_snapshot` → 获取当前战斗状态快照
  - `execute_action` → 执行指定游戏动作
  - `set_fast_mode` → 开启/关闭快速模式（跳过动画）
- **架构**：`ExternalControlService.cs`（C# TCP 监听）→ `BridgeAPI`（GDScript 节点，BattleManager 子节点）
- **Fast Mode**：训练模式下跳过所有动画延迟，加速数据采集。

### 13.2 训练集成

`training/` 目录包含 Python 训练脚本：
- `game_bridge.py` — Python TCP 客户端，封装游戏通信

### 14.1 地图生成规则

#### 14.1.1 章节结构
每章由 **4层节点** 组成：
- **Layer 0**（底层）：2-3个节点
- **Layer 1**（中层）：2-3个节点
- **Layer 2**（高层）：2-3个节点
- **Layer 3**（顶层）：1个Boss节点

每层配置：第一章 `[2, 3, 2]`，第二章 `[2, 3, 2]`。

#### 14.1.2 节点类型权重
| 类型 | 权重 | 说明 |
|------|------|------|
| 普通战斗 ⚔️ | 40% | 从普通敌人池随机抽取 |
| 精英战斗 ⚡ | 15% | 从精英敌人池随机抽取 |
| 事件 ❓ | 25% | 从7种事件池随机抽取 |
| 商店 🛒 | 10% | 6种商品可供购买 |
| 休息点 🏕️ | 10% | 双选项：恢复HP 或 注入EP |
| Boss 👹 | 固定 | 每章固定Boss |

**生成规则**：
- 每层至少保证1个普通战斗节点
- 其余节点按权重随机分配
- 节点间通过连接线连通（全连接：每个上层节点连接所有下层节点）
- 使用种子确保可复现性

#### 14.1.3 敌人池分配
| 章节 | 普通池 | 精英池 | Boss |
|------|--------|--------|------|
| 第一章 | 防火墙哨兵, 脉冲中继器, 数据腐化体 | 加密守护者, 虚空信标 | 湮灭协议·原型 |
| 第二章 | 内存吞噬者, 脉冲中继器·改, 数据腐化体·改 | 矩阵哨卫, 加密守护者·改 | 格式化巨兽 |

**敌人不重复规则**：同类型敌人在池耗尽前不重复抽取。池耗尽后重置。

### 14.2 节点交互流程

#### 14.2.1 普通/精英/Boss 战斗节点
1. 点击可用战斗节点
2. `RunRewardState.pending_battle` 设置战斗配置（enemy_key, node_type, is_elite）
3. 切换到 `battle.tscn` 场景
4. 战斗结束后返回地图，根据结果：
   - **胜利**：标记节点完成，解锁下一层，获得碎片奖励，弹出奖励选择
   - **失败**：显示游戏结束界面

#### 14.2.2 事件节点
点击后弹出事件面板，显示：
- 事件标题和故事描述（RichTextLabel，支持BBCode）
- 2-3个选择按钮
- 选择后执行效果，显示结果文本，关闭面板，标记节点完成

**7种事件详情**：

| 事件 | 标题 | 选项 |
|------|------|------|
| abandoned_database | 废弃数据库 | ① 随机卡牌+8EP ② +20碎片 |
| black_market | 黑市商人 | ① -25碎片,永久+1EP/回合 ② -15碎片,恢复20HP ③ +10碎片,下场敌人+2力量 |
| clone_pod | 故障克隆舱 | ① 复制一张卡牌 ② 随机+4EP ③ -5HP,+15碎片 |
| data_storm | 数据风暴 | ① -10HP,储备池+5 ② -8HP,+30碎片 |
| wandering_ai | 游荡AI | ① 永久移除一张卡牌 ② 随机+8EP |
| abandoned_armory | 废弃军械库 | ① 下场战斗+3EP/回合 ② +1能量护盾 |
| mysterious_altar | 神秘祭坛 | ① -10HP,随机+12EP ② -20HP,永久+1EP/回合 ③ 离开 |

#### 14.2.3 商店节点
点击后弹出商店面板，显示6种商品（2行×3列），每种商品可购买多次（只要碎片足够）：

| 商品 | 图标 | 价格 | 效果 |
|------|------|------|------|
| 进化模块 | 🧬 | 60 | 选择一张卡牌+8EP注入进度 |
| 纳米修复 | ❤️ | 40 | 恢复20HP |
| 随机卡牌 | 🃏 | 50 | 获得一张随机Lv.1卡牌 |
| 移除卡牌 | 🗑️ | 70 | 永久移除一张卡牌 |
| 能量饮料 | ⚡ | 30 | 下场战斗+2EP/回合 |
| 储备增压 | 🔋 | 45 | EP储备池+8进度 |

底部"离开商店"按钮关闭面板。

#### 14.2.4 休息点节点
点击后弹出休息面板，提供两个选项：
1. **恢复 15 HP** — 立即恢复15点HP
2. **随机卡牌 +4 EP** — 随机选择一个未满级的卡牌家族，注入4EP进度

### 14.4 章节过渡流程

1. **第一章Boss击败** → 弹出奖励选择 → 弹出核心突破协议选择（3选1）
2. 选择协议后：
   - 协议效果立即/持久生效
   - `RunRewardState.current_chapter` 设为 2
   - `MapGenerator` 生成第二章地图
   - 重新渲染地图界面
3. **第二章Boss击败** → 弹出奖励选择 → 显示通关成功界面
4. 通关界面显示统计：牌库大小、碎片数量、能量护盾层数、完全进化卡牌数
5. "返回主菜单"按钮重置Run状态并回到登录界面

### 14.5 全局行动日志

`ActionLog` Autoload 单例在整个Run期间记录所有操作：
- 战斗中的出牌、注入、进化、伤害等
- 地图中的节点选择、事件选项、商店购买等
- 每条日志包含时间戳（MM:SS格式）、文本（支持BBCode）、颜色
- 最多保存100条记录
- 战斗HUD和地图界面都有日志面板，实时显示新条目
- 新Run开始时清空日志

### 14.6 地图UI交互细节

#### 节点状态视觉
| 状态 | 背景色 | 边框色 | 边框宽度 | 交互 |
|------|--------|--------|----------|------|
| 已完成 | `#0A2A0A` | `#3BFF8C` 绿色 | 4px | 禁用，显示绿色图标 |

#### 连接线
- 外发光线：8px宽，`#5A5AFF` 20%透明度
- 主线：3px宽，`#7A7AFF` 85%透明度
- 中点圆标记：3.5px半径

#### 悬停效果
- 边框变为金色 `#FFD700`，5px宽
- Tooltip显示节点类型名称和敌人名称（如可用）

### 14.7 技术实现

#### MapGenerator（RefCounted）
```gdscript
# 核心方法
func generate_chapter(chapter: int, seed: int) -> Dictionary
func _assign_node_types(count: int, rng) -> Array  # 权重随机，保证至少1战斗
func _pick_enemy(difficulty, chapter, rng, used) -> String  # 不重复抽取
```

#### MapScreen（Control）
```gdscript
# 核心方法
func _render_map()  # 渲染节点和连接线
func _on_node_clicked(layer, index)  # 节点点击处理
func _enter_battle(node)  # 进入战斗
func _show_event_overlay(node)  # 事件面板
func _show_shop_overlay(node)  # 商店面板
func _show_rest_overlay(node)  # 休息面板
func _handle_return_from_battle()  # 战斗返回处理
func _show_core_breakthrough()  # 核心突破选择
func _show_victory_screen()  # 通关界面
func _show_game_over()  # 游戏结束界面
```

#### 场景切换数据流
```
login_screen → [点击开始] → RunRewardState.start_new_run()
                           → MapGenerator.generate_chapter()
                           → get_tree().change_scene_to_file("map_screen.tscn")

map_screen → [点击战斗节点] → RunRewardState.pending_battle = {...}
                            → get_tree().change_scene_to_file("battle.tscn")

battle → [战斗胜利] → RunRewardState.sync_from_ep_manager()
                    → RunRewardState.sync_hp_from_player()
                    → RunRewardState.add_fragments()
                    → get_tree().change_scene_to_file("map_screen.tscn")

map_screen → [Boss defeated] → _show_core_breakthrough()
                             → RunRewardState.current_chapter = 2
                             → MapGenerator.generate_chapter(2)
                             → _render_map()
```
